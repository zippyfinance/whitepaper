CHALLENGES IN BLOCKCHAIN

  • TPS: The bottleneck of blockchain:: the slow transaction speed
  • Security Miner Extractable Value: MEV-bots generate around 50% of blocks in Ethereum
  • Cost: Gas Fees become increasingly expensive for Ethereum

WE CAN HANDLE #1 High TPS

  • Loadsharing Sharding
    • Each shard has its own shard chain
    • Each block is evenly divided into chunks for different shards
    • Allow inter-shard communication
    • More shards, higher speed
  • Parallel EVM
    • Parallel transaction assignments into different chunks
    • Dependency of transactions is appended as metadata to blocks
    • Parallel confirmations on each shard chain
    • More parallel confirms, higher speed
  • Fast Consensus Mechanism
    • fBFT consensus mechanism
    • BLS multi-signature scheme
    • Complexity grows linearly in the number of nodes

WE CAN HANDLE #2 MEV Resistant & Secure

  • PBS (Proposer & Builder Scheme)
    • Proposers aggregate transactions into chunks in parallel
    • Builders confirm chunks with the highest gas fee
    • Similar sender addresses are collected into same chunks to avoid conflict in parallel execution
  • Loadsharing Sharding
    • Transactions confirmed in chunks
    • Chunk size decreases in the number of shards
    • More shard, less MEV
  • Security
    • The blockchain is secured by proof of stake (PoS)
    • The validators in each shard are randomly assigned using VDF+VRF
    • The validators will be reassigned each epoch, e.g., 20 hours
    • Limited voting power in each shard for a validator
